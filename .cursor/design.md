# 🌌 QWCard 前端设计与交互规范（更新版）

## 1. 总体风格定义
- **主基调**：白色为底，内容对比优先，科技感仅作点缀，绝不喧宾夺主。  
- **视觉特征**：低饱和、低不透明度、细线条；留白优先，字号层级清晰。  
- **科技感装饰**：极轻量，仅限 Header、时间线、分隔、Logo 墙等位置。  
- **交互氛围**：干净、简约、现代，保持“轻科幻”调性。  

### 字体与排版
- **推荐字体**：现代无衬线字体（Inter / 思源黑体 / HarmonyOS Sans / SF Pro）。  
- **层级清晰**：  
  - 标题（大字号，粗体）  
  - 小标题（中字号，中等权重）  
  - 正文（常规字号，正常权重）  
- **对比**：字号差异明显，保证信息优先级清晰。  

### 图标与色彩
- **图标风格**：线条细，圆角统一。  
- **配色**：随主题色走，默认科技蓝/灰/绿色调。  

---

## 2. 动效与可访问性
- **动效定位**：少而精，仅在背景/分隔/时间线/Logo 轮播承载。正文区域禁止动态干扰。  
- **实现方式**：仅用 `transform` / `opacity`，避免性能消耗；合成层可控。  
- **动效档位**：  
  - `reduced`：全部静态，适合弱机型/移动端。  
  - `default`（默认）：Header 网格/光晕 + 时间线流光 + Logo 轮播。  
  - `rich`：桌面端增强，增加光点/微扫描线。  
- **系统偏好**：支持 `prefers-reduced-motion`，自动降级。  
- **配置开关**（可租户定制）：  
  - `headerGlow`（头部光晕）  
  - `scanLine`（扫描线）  
  - `lineShimmer`（分隔/时间线流光）  
  - `logoMarquee`（Logo 墙慢速滚动）

## 3. 模块设计规范

### 模块基础规则
- **模块布局**：每个模块带有 `layout` 类型，可配置为：  
  - 图文左右排布  
  - 多图滑动  
  - 卡片栅格  
  - 时间线  
  - Logo 滚动  
- **模块可配置**：  
  - 名称、内容、顺序均由租户后台自由配置；  
  - 默认提供 4-5 个常用模板，支持扩展。  
- **样式**：  
  - 容器为圆角矩形卡片，浅阴影；  
  - 增加浅色描边或 1px 内阴影，提升边界精致感；  
  - 固定高度，进入详情页展示完整内容。  

### 模块类型（模板化可复用）
1. **企业/业务介绍**  
   - 标题+正文，支持文字、图文混排。  
2. **产品展示**  
   - 单行 3-4 张图片，点击进入详情；  
   - 或横向滑动查看更多。  
3. **方案介绍**  
   - 左右布局（左文字，右图片卡片）。  
4. **环境/工厂展示**  
   - 横向滑动图片流，放在富文本容器中。  
5. **时间线**  
   - 中线虚线 + 弱流光（10s 一次，2s 停驻），移动端静态降级。  
6. **Logo 墙**  
   - 30s 无缝慢速滚动，hover 暂停，尺寸与留白统一。

## 4. 推送卡片信息（消息预览规范）
- **推送类型**：  
  - **名片推送**（企业名片/个人名片）  
  - **素材推送**（素材库中的文档、图片、视频等）  
- **卡片样式**：统一圆角卡片，留白充足，浅描边，信息对比明显。  
- **可调整内容**：  
  - 标题（默认取模块名或素材标题，可手动修改）  
  - 描述摘要（自动提取前几行，可手动替换）  
  - 封面图（支持上传或从素材中选择）  
- **卡片预览规则**：  
  - 发送前可二次确认和编辑。  
  - 保持“白底+轻科幻”风格一致。  
  - 信息层级不超过三层（标题/摘要/封面）。

## 6. 性能与安全
- **性能优化**：仅启用轻量动画，不使用全局粒子，避免渲染阻塞。  
- **CSP 策略**：开发环境允许 HMR，生产环境不依赖 `eval`。  
- **可维护性**：所有模块模板化，支持替换/重命名/扩展。
